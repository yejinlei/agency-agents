# Examples

此目录包含示例输出 demonstrating how the agency's agents 可以被编排在一起来解决现实世界的任务.

## 为什么存在这个

agency-agents 仓库定义了几十个专业代理 across engineering, design, marketing, product, support, spatial computing, and project management. But agent definitions alone don't show what happens when you **deploy them all at once** on a single mission.

这些示例回答了问题: *"当整个代理协同工作时实际看起来是什么样的？"*

## 内容

### [nexus-spatial-discovery.md](./nexus-spatial-discovery.md)

**What:** 一个完整的产品发现练习 8 个代理并行工作 来评估软件机会 并产生一个统一计划.

**场景：** 网络研究确定了一个机会 在 AI 代理编排和空间计算的交叉点. 然后整个代理被同时部署 以产生：

- 市场验证和竞争分析
- 技术架构 (8 服务系统设计 带有完整 SQL 模式)
- 品牌战略和视觉识别
- 上市和增长计划
- 客户支持运营蓝图
- 带有用户画像和旅程图的 UX 研究计划
- 带有 65 个冲刺工单的 35 周项目执行计划
- 空间界面架构规范

**使用的代理：**
| Agent | Role |
|-------|------|
| Product Trend Researcher | 市场验证，竞争格局 |
| Backend Architect | 系统架构，数据模型，API 设计 |
| Brand Guardian | 定位，视觉识别，命名 |
| 增长 Hacker | GTM 战略，定价，发布计划 |
| Support Responder | 支持层级，入职，社区 |
| 用户体验研究er | Personas, 旅程图s, design principles |
| Project Shepherd | 阶段计划，冲刺，风险登记册 |
| XR Interface Architect | 空间 UI 规范 |

**关键要点：** 所有 8 个代理并行运行 并产生了连贯的、交叉引用的计划 无需协调开销. The output demonstrates the agency's ability to go from "find an opportunity" to "here's the full blueprint" in a single session.
